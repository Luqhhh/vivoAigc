# CodeMotion 演示脚本

演示基线：版本 `1.0.0`，优先使用 `LLM_MODE=mock` 保证确定性。当前阶段没有 APK artifact，现场打开 Web；若后续按部署教程构建并验证 APK，可使用相同界面替代 Web，但不得在未验证时声称 APK 已交付。

## 演示前检查

执行目录：仓库根目录。

```powershell
pnpm dev:api
```

另开窗口。执行目录：仓库根目录。

```powershell
pnpm dev:web
```

演示前完成：

- 打开 <http://localhost:5173>，确认服务状态为“模拟服务”或预期的“真实服务”。
- 打开 <http://localhost:3000/api/health>，确认 `ok: true`。
- 预先载入一次斐波那契和二分查找，确认视图与播放控制可用。
- 关闭通知、无关标签页和可能暴露凭据的终端内容。
- 不展示 `.env`、AppKEY、Authorization header 或未脱敏日志。

## 3 分钟主流程

| 时间 | 操作 | 讲解词 |
| --- | --- | --- |
| 00:00-00:15 | 打开 CodeMotion Web 工作台。 | “CodeMotion 把 Python 算法的执行过程变成可以逐步观察和追问的学习界面，不需要登录，也不会在浏览器执行用户代码。” |
| 00:15-00:35 | 点击底部“示例”，载入“斐波那契递归”，自动回到工作台。 | “先用一个稳定示例演示。代码、知识点和推荐视图都已准备好，评审可以直接进入学习流程。” |
| 00:35-00:55 | 点击“分析代码”，等待结果；指向来源标签、摘要和复杂度。 | “后端返回结构化分析。这里明确显示结果来源；本次演示使用 mock，绝不把模拟结果说成真实蓝心调用。” |
| 00:55-01:25 | 点击播放，再暂停；使用下一步展示代码高亮、变量和步骤解释同步变化。 | “当前步骤是唯一播放状态，代码行、变量和解释随它同步。播放到最后会自动停止。” |
| 01:25-01:55 | 依次切换“调用栈”和“递归树”，点击一个递归节点跳到对应步骤。 | “调用栈说明函数正在等待谁，递归树显示 `fib(4)` 如何拆成子调用；切换视图不会丢失当前步骤。” |
| 01:55-02:25 | 定位到 `fib(2)` 相关步骤，在 AI 导师输入“为什么 fib(2) 会返回 1？”，发送并点击引用步骤。 | “导师收到当前代码、分析摘要和当前步骤，因此回答能落回这一段执行上下文，而不是泛泛解释递归。” |
| 02:25-02:45 | 点击底部“练习”，展示至少三条相关建议。 | “分析结果继续生成相似练习，每条包含难度、知识点、推荐理由和可视化提示。” |
| 02:45-03:00 | 打开“作品”页，指出 Web/API 和适配器状态。 | “架构把凭据限制在 API 服务端。真实服务失败时会明确标为 fallback；本阶段交付 Web、API 和完整 APK 教程，不宣称已有 APK artifact。” |

### 3 分钟失败降级

| 失败 | 立即操作 | 讲解词 |
| --- | --- | --- |
| real Lanxin 不可用或超时 | 指向“降级分析”与 warning；继续播放稳定结果。 | “真实链路失败后没有白屏，60 秒后超时并显式 fallback。Fluid Compute 必须保持启用，且函数执行时长必须超过 60 秒 provider 超时。为了演示连续性，下面使用确定性结果。” |
| API 未启动 | 启动 `pnpm dev:api`；确认健康检查成功后刷新 Web 页面。 | “前端会显示服务离线，不暴露上游错误；服务恢复并刷新页面后可重试。” |
| 分析请求失败 | 点击“重新分析”；仍失败则载入斐波那契示例并切回 mock。 | “错误状态可恢复，示例和 mock 是现场保底路径。” |
| 递归树缺失 | 回到时间轴与调用栈，使用内置斐波那契示例重试。 | “递归树只在响应包含合格递归结构时展示，其他算法仍可用时间轴和调用栈解释。” |
| 导师请求失败 | 使用当前步骤解释和行说明完成讲解。 | “导师是增强能力，核心 trace、变量和调用栈不依赖这次追问成功。” |
| 时间不足 | 跳过“作品”页，口头说明 source 和交付边界。 | “结果来源可追踪，凭据只在后端；当前无 APK artifact。” |

| 03:10-03:45 | 恢复默认代码或载入斐波那契，再点击分析。 | “除了示例，也可以输入不超过 200 行、12,000 字符的 Python 片段。后端只分析文本，不执行代码。” |

| 05:20-05:55 | 打开“练习”页，展示推荐；点击返回工作台。 | “相似练习延续刚学的概念，不复制第三方完整题面，并保留当前代码上下文。” |

| 07:40-08:00 | 回到工作台，停在递归树或二分变量画面。 | “CodeMotion 的核心价值是把 AI 解释绑定到可检查的执行步骤，并保留 mock 降级和证据化验收路径。” |

### 8 分钟失败降级

1. **开场前 API 异常**：立即切到 mock 配置并重启 API；若仍不可用，展示前端离线状态、已有界面和部署文档，不伪造在线分析。
2. **real 返回 fallback**：保留 warning 和 `source: fallback` 画面，说明真实 gateway 未通过本次调用；继续完成可视化流程。
3. **二分示例异常**：切换斐波那契示例，先保证时间轴、调用栈和递归树主价值可见；变量变化用预先保存且脱敏的 Task 8 证据补充，当前没有证据时不得声称通过。
4. **导师异常**：改用步骤解释、变量和行说明；明确导师请求失败，不把静态文案说成实时回答。
5. **生产网络不可用**：使用本地 Web + API；健康检查使用 `localhost`，不声称生产 URL 可访问。
6. **Android 工具链不可用**：只展示 APK 教程和 `PENDING / NOT EXECUTED` 测试项，不展示占位 APK 或虚构签名结果。
7. **时间超支**：优先保留二分变量、斐波那契递归树、导师追问、`source` 和安全边界；压缩架构与部署口述，不跳过交付边界声明。
